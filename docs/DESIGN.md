# SightMate 设计文档

当前进度：PR 1 初始化项目。本文件会在后续 PR 中随功能完善。

## 用户故事

### 计划实现

1. 作为用户，我希望打开摄像头后，AI 可以理解当前画面。
2. 作为用户，我希望通过语音向 AI 提问。
3. 作为用户，我希望 AI 能用语音回答我。
4. 作为用户，我希望看到历史对话记录。
5. 作为用户，我希望系统不要持续上传视频流，保护隐私。

### 最终实现

- 已实现：项目基础结构。
- 已实现：后端健康检查接口。
- 未完成：摄像头预览、语音输入、视觉问答、语音播报、历史记录和完整隐私提示，后续 PR 继续实现。

### 未完成原因

当前任务书要求只完成 PR 1：初始化项目。

## 成本控制策略

### 计划考虑的策略

1. 不上传连续视频流。
2. 用户提问时才截图。
3. 压缩图片后再上传。
4. 只保留最近 3 轮上下文。
5. 优先使用浏览器语音识别和语音播报。
6. 增加按钮 loading，避免重复调用。
7. 使用 localStorage 保存历史，避免引入云数据库。

### 实际采用的策略

当前 PR 只完成项目初始化，尚未产生模型调用成本。项目结构已经预留前后端分离架构，后续所有模型请求都会通过后端统一处理，避免前端暴露 API Key。
