# SightMate 设计文档

## 用户故事

### 计划实现

1. 作为用户，我希望打开摄像头后，AI 可以理解当前画面。
2. 作为用户，我希望通过语音向 AI 提问。
3. 作为用户，我希望在语音识别不可用时仍能用文字提问。
4. 作为用户，我希望 AI 能用文字回答我。
5. 作为用户，我希望 AI 回答后可以自动语音播报。
6. 作为用户，我希望看到历史对话记录，包括问题、回答、截图和时间。
7. 作为用户，我希望可以清空历史记录。
8. 作为用户，我希望系统不要持续上传视频流，保护隐私。
9. 作为开发者，我希望前端不直接请求阿里云 API，避免暴露 API Key。

### 最终实现

- 已实现：摄像头实时预览。
- 已实现：浏览器语音识别输入。
- 已实现：文字输入兜底。
- 已实现：发送问题时自动截取当前画面。
- 已实现：截图压缩为 JPEG base64，最长边不超过 1024px。
- 已实现：前端发送问题、截图和最近上下文到后端。
- 已实现：后端调用阿里云百炼千问视觉模型。
- 已实现：前端展示 AI 回答。
- 已实现：AI 回答后自动语音播报。
- 已实现：停止播报和重新播放。
- 已实现：localStorage 保存最近 20 条历史记录。
- 已实现：发送请求时只带最近 3 轮上下文。
- 已实现：清空历史记录。
- 已实现：loading、防重复提交、错误提示和隐私提示。

### 未完成的用户故事及原因

- 未实现：云端账号体系和跨设备历史同步。原因是任务书要求使用 localStorage 保存历史，避免引入云数据库和额外成本。
- 未实现：持续视频流上传和实时视频通话式 AI。原因是任务书明确要求不做持续视频流上传，只在用户主动提问时截取当前帧。
- 未实现：阿里云智能语音交互。原因是任务书要求优先使用浏览器 Web Speech API 和 `speechSynthesis`，云端语音服务作为后续扩展。

## 成本控制策略

### 计划考虑的策略

1. 不上传连续视频流。
2. 用户主动提问时才截图。
3. 压缩图片后再上传。
4. 只保留最近 3 轮上下文用于模型请求。
5. 优先使用浏览器语音识别和语音播报。
6. 增加按钮 loading，避免重复调用。
7. 使用 localStorage 保存历史，避免引入云数据库。
8. 前端不直接请求模型服务，统一由后端管理 API Key 和模型调用。

### 实际采用的策略

1. **事件触发式截图**：只有点击发送问题时才截取当前画面，避免持续视频流成本。
2. **图片压缩**：前端使用 canvas 将截图转成 JPEG base64，最长边限制为 1024px，减少模型输入体积。
3. **上下文裁剪**：历史记录可保存 20 条，但每次请求只发送最近 3 轮上下文，控制 token 成本。
4. **浏览器语音能力**：语音识别使用 Web Speech API，语音播报使用 `speechSynthesis`，不额外调用云端语音服务。
5. **本地历史存储**：使用 localStorage 保存对话历史，不部署数据库。
6. **防重复提交**：模型请求 loading 期间禁用发送按钮，避免用户连续点击造成重复调用。
7. **后端代理模型请求**：所有阿里云模型请求经过 FastAPI 后端，前端不暴露 API Key，便于后续统一做限流、日志和错误处理。

### 为什么能降低成本

- 视频流通常意味着持续带宽和持续模型推理，按需截图只在用户真正提问时产生一次模型调用。
- 图片越小，上传耗时和多模态模型输入成本越可控。
- 上下文越短，文本 token 成本越低，响应速度也更稳定。
- 浏览器语音能力不产生额外云服务调用费用。
- localStorage 避免数据库、对象存储和后端历史查询接口成本。
- 防重复提交可以减少误触导致的重复模型调用。

## 核心流程

```text
打开摄像头 -> 语音或文字提问 -> 发送时截取当前帧 -> 前端调用后端接口 -> 后端调用千问视觉模型 -> 返回回答 -> 前端展示并语音播报 -> 保存本地历史
```

## 接口设计

### GET /health

返回：

```json
{
  "status": "ok"
}
```

### POST /api/vision/chat

请求：

```json
{
  "question": "请描述一下当前画面",
  "image_base64": "data:image/jpeg;base64,...",
  "history": [
    {
      "role": "user",
      "content": "画面里有什么？"
    },
    {
      "role": "assistant",
      "content": "画面里有一台电脑和一个水杯。"
    }
  ]
}
```

响应：

```json
{
  "answer": "当前画面中可以看到一台电脑、一个水杯和一些桌面物品。",
  "model": "qwen-vl-plus",
  "created_at": "2026-06-12 20:00:00"
}
```

## 隐私设计

- 摄像头实时预览只在浏览器本地展示。
- 不上传连续视频流。
- 只有用户发送问题时才上传单张截图。
- 真实 API Key 只保存在后端 `.env` 中。
- 历史记录默认保存在当前浏览器 localStorage。
